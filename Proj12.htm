PROGRAM SPECIFICATIONS

The zip file that you downloaded contains a dataset file named data.csv.

You must submit a Python file named Proj12Runner.py and any other necessary Python source code files to run under control of the driver source code contained in the file named Proj12.py. Your code must read the data from the file named data.csv and produce two items of screen output. Example screen output is shown in Case 1, Case 2, Case 3, and Case 4 below.

The csv data file must be in the same folder as the Python source code files when the program is executed.

The screen output for each case consists of two parts. One part is a graph that appears on the screen. See Figure 1b for example. The other part is text that appears on the command-line screen or the system console. See Figure 1a for example. The output on the command-line screen shows a variety of results based on the contents of the dataset and the number of rows of data being plotted. The name of the data file and the number of rows of data being plotted are specified by command-line arguments when the program is executed as explained later.

The program must terminate and return control to the operating system when control reaches the end of the file named Proj12.py. In other words, infinite loops are not allowed.

Do not modify the code in the file named Proj12.py. Do not copy the code from the file named Proj12.py into your file named Proj12Runner.py.

The script in the file named Proj12.py requires that you enter two command-line arguments separated by a comma in the following format:

data.csv,(Some positive numeric value less than 121.)

The first command-line argument specifies the name of the data file. The second command-line argument specifies how many rows of data are to be plotted after collapsing rows with duplicate dates as discussed later.

In all cases, the rows of data that are to be plotted must include data from the first row of data in the raw dataset that looks like the following:

"21 Sep 2022","3:50 pm","126","72","64","-","-"

You need to include code to rename two of the data columns as shown below:

Change Systolic (mmHg) to Syst
Change Diastolic (mmHg) to Dias

The dataset doesn't contain a row for every date. Thus there are gaps in the data and those gaps must be reflected in the graph.

In some cases, the dataset may contain multiple rows for the same date. In those cases, each group of multiple rows having the same date must be collapsed into a single row with the data values for that single row being the mean of the values for the multiple rows in the group.

The required output depends on the command-line arguments that are entered when the program is run. Four examples with argument values of 31, 61, 91, and 111 are shown in Case 1, Case 2, Case 3, and Case 4 below. You can use those cases to test your program.

You should also test your program with other cases as well to ensure consistent behavior regardless of the number of rows. Your program must not fail for any numeric value in the second command-line argument, but need not produce meaningful results for negative arguments, arguments less than ten, or arguments greater than 120.

Your instructor will use a different command-line argument specifying the number of rows to plot than any of those shown below to test your program.

Determining the correct tick marks and labels can be challenging. It might be helpful for you to print the data you intend to plot and compare the tick labels in the examples with the date values in the data. This process can help you deduce the algorithm required for selecting and displaying tick marks and labels. Be aware that Pythonâ€™s handling of truncation in integer division may cause small variations in the specific dates chosen as tick marks.

The graphs that you create must be date or calendar sensitive. By this I mean that the positions of the tick marks on the horizontal axis must represent sequential days of the year. As illustrated by the left end of the plot shown in Figure 2b, there must be a horizontal gap for each day for which there is no data.

Each data point must be represented by an "o" marker with a marker size of 4. The data points must be connected by straight lines. There must be no data points plotted for those days for which there is no data.

The graph must plot three curves with different colors. Each curve is a plot of the data from one of the columns in the csv file. The colors aren't critical as long as they are different. Let the system decide which colors to use.

Each curve must begin and end on a data point and a vertical grid line. You should test your program against that requirement by running your code with a variety of command-line arguments between 10 and 120 inclusive.

There must be a legend in the center as shown in the examples below.

The horizontal tick marks and their corresponding vertical grid lines must align precisely with the data points. The number of data points must equal the number of rows being plotted. The total number of tick marks and vertical grid lines should be calculated as the integer result of dividing the number of rows by 5. As a result of truncation during integer division, the spacing between vertical grid lines will typically contain either five or six data points. The x-axis tick labels must match the values and positions shown in all four examples provided below. Avoid hardcoding specific dates for the tick labels in your code. Instead, implement a generalized solution that dynamically calculates and displays accurate tick marks and labels for any number of rows used during testing.
